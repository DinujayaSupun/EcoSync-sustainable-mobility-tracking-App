## Running Tests

### Prerequisites
```bash
npm install
```

Ensure these dependencies are installed:
- `jest` - Test framework
- `supertest` - HTTP assertions for API testing
- `mongodb-memory-server` (optional) - In-memory MongoDB for isolated testing

### Environment Variables
Create `.env.test` file or set these variables:
```bash
# Required
MONGODB_URI_TEST=mongodb://localhost:27017/sustainability_test
JWT_SECRET=your-test-jwt-secret

# Optional
SUPPRESS_TEST_LOGS=true
```

### Run All Tests
```bash
npm test
```

### Run Specific Test File
```bash
npm test admin.controller.test.js
npm test activityLog.test.js
```

### Run with Coverage Report
```bash
npm test -- --coverage
```

### Run in Watch Mode
```bash
npm test -- --watch
```

### Run Verbose Output
```bash
npm test -- --verbose
```

## Troubleshooting

### Tests Fail to Connect to Database
- Ensure MongoDB is running
- Check `MONGODB_URI_TEST` environment variable
- Consider using `mongodb-memory-server` for isolated tests

### Rate Limiting Tests Fail
- Rate limiters may need to be mocked in test environment
- Check if rate limit configuration uses different settings for testing

### Timeout Errors
- Increase timeout in `jest.config.js`
- Check database connection speed
- Ensure no hanging processes

### Authentication Errors
- Verify `JWT_SECRET` is set
- Check token generation logic
- Ensure admin user is created in beforeEach

## Best Practices

1. **Isolation**: Each test is independent with clean database before/after
2. **Performance**: Tests should complete in < 10 seconds
3. **Coverage**: Aim for > 80% code coverage
4. **Clarity**: Test names clearly describe what they verify
5. **Maintenance**: Update tests when changing business logic

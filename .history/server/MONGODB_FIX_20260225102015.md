## 🚨 CRITICAL: Fix IP Whitelist Issue

Since you've tried multiple times, follow these steps **carefully**:

### Option 1: Allow Access from Anywhere (RECOMMENDED for Development)

1. Go to [MongoDB Atlas](https://cloud.mongodb.com/)
2. Sign in to your account
3. Select your project
4. Click **"Network Access"** in the left sidebar (under Security)
5. Click **"ADD IP ADDRESS"** button
6. Click **"ALLOW ACCESS FROM ANYWHERE"**
7. It will automatically fill: `0.0.0.0/0`
8. Add a comment like "Development Access"
9. Click **"Confirm"**
10. ⏱️ **WAIT 2-3 MINUTES** for changes to propagate

### Option 2: Add Your Specific IP

1. Find your current IP: Visit https://whatismyipaddress.com/
2. In MongoDB Atlas → **Network Access** → **ADD IP ADDRESS**
3. Click **"ADD CURRENT IP ADDRESS"**
4. Click **"Confirm"**
5. ⏱️ **WAIT 2-3 MINUTES**

## 🔑 Verify Your Credentials

1. Go to MongoDB Atlas → **Database Access** (under Security)
2. Verify you have a user named: `afNew`
3. If not, create one:
   - Click **"ADD NEW DATABASE USER"**
   - Username: `afNew`
   - Password: `afNew` (or whatever you want)
   - Database User Privileges: **"Atlas admin"** or **"Read and write to any database"**
   - Click **"Add User"**

## 🔄 If You Need to Reset Connection String

1. Go to MongoDB Atlas → **Database** (left sidebar)
2. Click your cluster name (Cluster0)
3. Click **"Connect"** button
4. Select **"Drivers"**
5. Copy the connection string
6. Replace in your `.env` file:
   ```
   MONGO_URI=mongodb+srv://<username>:<password>@cluster0.fz1a1au.mongodb.net/sustainabilityDB?retryWrites=true&w=majority&appName=Cluster0
   ```
7. Replace `<username>` with `afNew`
8. Replace `<password>` with `afNew`

## 🆘 Still Not Working?

### Check 1: Cluster Status

- Go to Atlas → Database
- Ensure your cluster shows "Active" (not paused)

### Check 2: Firewall/Antivirus

- Temporarily disable Windows Firewall
- Try again

### Check 3: Use MongoDB Compass

- Download [MongoDB Compass](https://www.mongodb.com/products/compass)
- Paste your connection string
- Try to connect
- If Compass can't connect, it's definitely an Atlas config issue

### Check 4: VPN/Proxy

- If you're using a VPN, try disconnecting
- Corporate networks might block MongoDB Atlas (port 27017)
